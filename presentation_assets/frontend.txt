Frontend Documentation - Virtual Fitting Room
English frontend report covering technologies, implementation decisions, problems faced, and solutions.
1. Frontend Overview
The Virtual Fitting Room frontend is built as part of an ASP.NET Core MVC application. Its purpose is to give users a clear flow
for signing in, uploading a model image and a clothing image, using the camera, and generating an AI-powered virtual try-on
result.
The frontend was improved from a generic AI-generated-looking interface into a more practical and professional product
experience. The current UI has clearer screens, realistic fashion visuals, consistent controls, and direct user journeys.

6. Fixing the AI-Generated Look
One major issue was that the first version of the website looked like a generic AI-generated template rather than a real product.
This was solved through several frontend changes:
Replaced generic visuals with realistic fashion-related imagery.
Reduced long promotional text and made copy more direct.
Used a more balanced color system instead of relying on excessive gradients.
Standardized buttons, inputs, cards, and spacing across pages.
Added practical UI details such as icons, image previews, camera controls, and bottom navigation.
Made the studio open directly into a useful camera-based experience instead of only showing a static landing layout.
Improved responsive behavior so the UI looks intentional on mobile screens.

7. Key Design Decisions
A dark visual theme was used to highlight user photos and garment images.
Glass-style cards were used on authentication screens to create a modern but focused look.
Icons were added to buttons to make actions easier to understand quickly.
The studio was designed with a mobile-app-like structure because camera usage is natural on phones.
Main actions were kept clear: Camera, Garment, Link, and AI Fit.
The Upload workflow was separated from the Live workflow to support two clear usage paths.
8. JavaScript Functionality
Show and hide password fields.
Open and close the camera.
Capture camera frames and convert them to Base64 images.
Show instant previews for selected images.
Resize images before submitting them.
Load MediaPipe Pose only when needed.
Store pose landmarks in hidden fields for backend processing.
Automatically match garment categories to fitting areas.
Prevent form submission when required images are missing.
Run AI Fit from the Live screen.
Open the history modal for previous try-on results.
9. Responsive Design
CSS Grid and Flexbox layouts.
Single-column layouts on mobile.
Adaptive camera and preview heights.
Fixed bottom navigation inside the studio.
Touch-friendly Live controls.
Rules that prevent text and UI elements from overlapping.
10. Backend and AI Integration
The frontend sends data to ASP.NET MVC controllers through forms. Images can be sent as file uploads or as Base64 data
captured from the camera. The backend then connects to the virtual try-on service, which can run locally or through Hugging
Face / Colab depending on the configuration.
The submitted frontend data includes:
Model image.
Clothing image.
Target body.
Clothing type.
Fitting area.
Model view.
Pose landmarks when available.

12. Summary
The project frontend evolved from a generic-looking interface into a practical UI that matches the Virtual Fitting Room concept.
Razor, Bootstrap, CSS, and JavaScript were used with browser APIs such as Camera, Canvas, and MediaPipe Pose to create a real
user experience. The most important improvements were clearer user flow, better visual design, camera support, image previews,
live fitting behavior, and responsive mobile usability.
